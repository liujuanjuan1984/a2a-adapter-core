# AGENTS.md

以下规范适用于 `a2a-adapter-core` 仓库内的 coding agent 协作与交付流程。

## 1. 核心原则

- 在保证安全、可追踪的前提下推进任务。
- 保持与现有仓库结构、实现风格和工程习惯一致。

## 2. Git 工作流

- 禁止直接向受保护分支提交或推送：`main`/`master`/`release/*`。
- 每项开发任务应在独立分支实施。
- 仅提交本次任务相关文件。

## 3. 回归与验证

- 默认基线验证：`uv run pytest` 和 `uv run pre-commit run --all-files`。
- 在创建或更新 PR 之前，必须确保本地验证通过。

## 4. 安全与配置

- 严禁提交密钥、令牌、凭证或其他敏感信息。
- 日志与调试输出不得泄露访问令牌或隐私数据。
